# WordLoop

Приложение для изучения словарных слов методом интервального повторения на базе
модифицированного SM-2. Поддерживает уроки, список слов, озвучку и режим
случайного изучения без изменения данных.

## Возможности
- Уроки с тремя состояниями: вопрос, ответ, нет слов на сегодня.
- Озвучка слова через TextToSpeech, ожидание инициализации с индикатором.
- Проверка ответа с подсветкой ошибок (правильно/ошибка/пропуск).
- Список всех слов с добавлением, редактированием и удалением.
- Цветовая индикация статуса слова в списке.
- Режим случайного изучения без обновления БД.

## Алгоритм интервального повторения
Используется SuperMemo SM-2 с упрощенной оценкой качества ответа (q = 0 или 5).

- Формула пересчета интервала: `I(n) = I(n-1) * ef`
- Новый коэффициент легкости:
  `ef' = ef + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))`
- Нижняя граница коэффициента: `ef >= 1.3`

Правила обновления карточки:
- `q = 0`:
  - статус `PROGRESS_RESET`;
  - `ef` пересчитывается только при первом сбросе;
  - интервал = 1 сутки;
  - `next_review_date = now + 10 минут`.
- `q = 5` при статусах `NEW` или `PROGRESS_RESET`:
  - `ef` не пересчитывается;
  - интервал = 1 сутки;
  - `next_review_date = now + 1 день`.
- `q = 5` при статусе `IN_PROGRESS`:
  - пересчет `ef`, интервала и даты следующего повтора.

## Экран урока
Состояния:
- **Вопрос** — озвучка слова, ввод ответа, проверка.
- **Ответ** — показывается правильное слово и подсветка ошибок.
- **Нет слов** — сообщение и кнопка перехода в режим случайного изучения.

Особенности:
- Автофокус на поле ввода и открытие клавиатуры.
- Скрытие клавиатуры на экране ответа.
- Кнопка проверки отключена до инициализации TTS.
- Подсветка ответа основана на выравнивании строк (редакционное расстояние).

## Экран списка слов
Показывает все слова в алфавитном порядке.

Цвет фона строки:
- мягкий красный: `PROGRESS_RESET`;
- мягкий зеленый: повтор сегодня или в прошлом;
- мягкий синий: дата повтора в будущем.

После слова отображается количество дней до следующего повтора.

## Режим случайного изучения
Доступен, когда слова на сегодня закончились.

Поведение:
- выбирается случайное слово из БД;
- при проверке ответов карточки не обновляются;
- переход на следующее слово также случайный.

## Публикация в Google Play через GitHub Actions

Для проекта добавлен ручной workflow:
`/.github/workflows/play-internal-release.yml`

Он собирает подписанный `.aab` и загружает его в трек `internal` в Google Play.

### Требуемые GitHub Secrets

- `ANDROID_KEYSTORE_BASE64` — содержимое release keystore в base64.
- `ANDROID_KEYSTORE_PASSWORD` — пароль keystore.
- `ANDROID_KEY_ALIAS` — alias ключа подписи.
- `ANDROID_KEY_PASSWORD` — пароль ключа подписи.
- `PLAY_SERVICE_ACCOUNT_JSON_BASE64` — JSON сервисного аккаунта Google Play в base64.

### Как запустить релиз

1. Увеличьте `versionCode` в `app/build.gradle` перед релизом.
2. Откройте в GitHub: `Actions` -> `Play Internal Release`.
3. Нажмите `Run workflow`.
4. Дождитесь завершения job `publish-internal`.

### Что делает workflow

1. Декодирует keystore и service account JSON из secrets.
2. Генерирует `keystore.properties` для release-подписи.
3. Запускает `:app:bundleRelease`.
4. Запускает `:app:publishReleaseBundle` для трека `internal`.
5. Сохраняет `.aab` как build artifact.

### Проверка результата

1. Откройте Google Play Console.
2. Перейдите в приложение -> `Testing` -> `Internal testing`.
3. Убедитесь, что появился новый release с нужным `versionCode`.
